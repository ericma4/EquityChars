## Fix 1: Momentum Windows — `chmom`, `mom6m`, `mom36m`, `mom60m`, `seas1a`

**Location**: ~Line 2167-2174

**Changes**:
| Variable | Before | After | Rationale |
|----------|--------|-------|-----------|
| `chmom` | `chmom(0, 6)` | `chmom(1, 7)` | Skip lag 0 (current month); use 6-month windows (lags 1-6 vs 7-12) |
| `mom6m` | `mom(1, 6)` = lags 1..5 | `mom(1, 7)` = lags 1..6 | 6-month momentum = 6 returns (t-1 to t-6) |
| `mom36m` | `mom(13, 36)` = lags 13..35 | `mom(13, 37)` = lags 13..36 | GHZ: "return between t-36 and t-13" = 24 returns at lags 13..36 |
| `mom60m` | `mom(13, 60)` = lags 13..59 | `mom(13, 61)` = lags 13..60 | HXZ A.2.8: "prior returns from t-60 to t-13" = 48 returns at lags 13..60 |
| `seas1a` | `shift(11)` | `shift(12)` | Same calendar month last year = 12 months ago |

**References**:
- Jegadeesh & Titman (1993): Momentum strategies skip the most recent month to avoid short-term reversal (Jegadeesh 1990). Since this pipeline predicts `r_{t+1}`, momentum signals at time `t` should exclude `ret_t` (lag 0).
- GHZ (Mom12m): "Cumulative return from month t-12 to t-2" — `mom(1, 12)` was already correct.
- GHZ (Mom6m): "Return from t-7 to t-2" — 6 monthly returns at lags 1..6 = `mom(1, 7)`.
- GHZ (Mom36m): "Stock return between months t-36 and t-13" — lags 13..36 = `mom(13, 37)`.
- HXZ A.2.8 (De Bondt & Thaler 1985): "Prior returns from month t-60 to t-13" — lags 13..60 = `mom(13, 61)`.
- HXZ A.5.51 / Heston & Sadka (2008): "`Ra1` = returns in month t-12" — `shift(12)`.

## Fix 2: `seas1a` — Same Calendar Month Last Year

**Before**: `pl.col('ret').shift(11).over('permno').alias('seas1a')`
**After**: `pl.col('ret').shift(12).over('permno').alias('seas1a')`

## Fix 3: `indmom` — Wrong Return Window, Weighting, and Industry Classification

**Location**: ~Line 2389 (annual), ~Line 2489 (quarterly)

**Before**:
```python
df_temp = data_rawa.group_by(['date', 'ffi49']).agg(pl.col('mom12m').mean().alias('indmom'))
```

**After**:
```python
# Value-weighted 6-month return over 2-digit SIC industries
data_rawa = data_rawa.with_columns([(pl.col('sic') // 100).alias('sic2')])
data_rawa = data_rawa.with_columns([(pl.col('mom6m') * pl.col('me')).alias('_vw_mom6m')])
df_temp = (data_rawa
    .filter(pl.col('me').is_not_null() & pl.col('mom6m').is_not_null())
    .group_by(['date', 'sic2'])
    .agg([pl.col('_vw_mom6m').sum(), pl.col('me').sum().alias('_sum_me')])
    .with_columns([(pl.col('_vw_mom6m') / pl.col('_sum_me')).alias('indmom')])
    .select(['date', 'sic2', 'indmom'])
)
data_rawa = data_rawa.join(df_temp, on=['date', 'sic2'], how='left')
```

Three issues corrected:

| Issue | Before | After | Paper |
|-------|--------|-------|-------|
| Return window | `mom12m` (12-month) | `mom6m` (6-month) | GHZ: "six-month buy-and-hold return" |
| Weighting | `mean()` (equal-weighted) | Value-weighted by ME | GHZ: "weights are based on market value of equity" |
| Industry | `ffi49` (Fama-French 49) | `sic2` (2-digit SIC) | GHZ: "two-digit industries" |

**Reference**: Grinblatt & Moskowitz (1999), "Do Industries Explain Momentum?", JF. GHZ (IndMom).

### 关于 `ffi49` vs `sic2` 的讨论：能否继续用 ffi49？

**GHZ SignalDoc 原文定义** ([OpenSourceAP/CrossSection/SignalDoc.csv](https://github.com/OpenSourceAP/CrossSection/blob/master/SignalDoc.csv)):
> "Weighted average of firm-level **6 month buy-and-hold return**. Average is taken over **two digit industries** each month and weights are based on **market value of equity**."

**Grinblatt & Moskowitz (1999) 原文**:
> 使用 2-digit SIC codes 将股票分为约 20 个行业组合（经过合并），每个行业平均 230 只股票。

#### 两种行业分类的区别

| 维度 | 2-digit SIC | Fama-French 49 (ffi49) |
|------|-------------|------------------------|
| 行业数量 | ~60-80 个（原始），论文合并为 ~20 个 | 49 个 |
| 分类依据 | 标准行业分类代码（直接按 SIC 前两位分组） | Fama & French 根据经济相似性手工归类 SIC codes |
| 粒度 | 某些行业过大（如 SIC 20-29 制造业）或过小 | 经过平衡，每组规模更均匀 |
| 学术使用 | Grinblatt & Moskowitz 原文使用 | 实证资产定价中更常用 |
| 覆盖率 | 所有有 SIC 的公司 | 需要 SIC → FF49 映射，少数公司可能归入 "Other" |

#### 三个问题的重要性排序

**原始代码有三个偏差，严重程度不同：**

1. **Return window: `mom12m` → `mom6m`** — **最重要**。论文明确写了 "6 month buy-and-hold return"，用 12 个月的 window 会混入长期反转效应，信号含义完全不同。

2. **Weighting: EW → VW** — **重要**。论文明确说 "weights are based on market value of equity"。等权平均会让小盘股过度影响行业动量信号，与论文原意不符。

3. **Industry: sic2 → ffi49** — **最不重要**。这是两种行业分类方案的选择，都有学术依据，且两者相关性很高。Grinblatt & Moskowitz 原文用 SIC 是因为他们的研究目标是检验 "行业" 是否解释动量，用更细粒度的 SIC 分类更贴近这个问题。但在实际构造预测因子时，ffi49 是完全合理的替代方案——许多后续论文和复制研究都使用 FF 行业分类。

#### 结论

**可以继续使用 `ffi49`。** 行业分类的选择（sic2 vs ffi49）对 `indmom` 的影响远小于 return window 和 weighting 的修正。如果要严格复制 GHZ/Grinblatt & Moskowitz，应使用 `sic2`；但如果项目中其他 `_ia` 变量都用 `ffi49`（herf, chempia, chatoia, bm_ia 等），那么 `indmom` 也用 `ffi49` 保持一致性是合理的。

**已将代码改回 `ffi49` 版本（保留 VW 和 mom6m 修正）：**

```python
# indmom: value-weighted 6-month return over ffi49 industries
# (fix)2026-03-13: return window mom12m→mom6m, weighting EW→VW
# Industry kept as ffi49 for consistency with other _ia variables (bm_ia, chatoia, etc.)
data_rawa = data_rawa.with_columns([
    (pl.col('mom6m') * pl.col('me')).alias('_vw_mom6m')
])
df_temp = (data_rawa
    .filter(pl.col('me').is_not_null() & pl.col('mom6m').is_not_null())
    .group_by(['date', 'ffi49'])
    .agg([
        pl.col('_vw_mom6m').sum().alias('_sum_vw'),
        pl.col('me').sum().alias('_sum_me')
    ])
    .with_columns([
        (pl.col('_sum_vw') / pl.col('_sum_me')).alias('indmom')
    ])
    .select(['date', 'ffi49', 'indmom'])
)
data_rawa = data_rawa.join(df_temp, on=['date', 'ffi49'], how='left')
data_rawa = data_rawa.drop('_vw_mom6m')
```

## Fix 4: `cfp_ia` — Look-Ahead Bias in Industry Adjustment

**Location**: ~Line 2342 (annual), ~Line 2441 (quarterly)

**Before**: `data_rawa.group_by(['datadate', 'ffi49'])`
**After**: `data_rawa.group_by(['jdate', 'ffi49'])`

After the CRSP monthly merge, `data_rawa` is at monthly frequency. Using `datadate` (fiscal year-end) to form industry cross-sections groups firms whose financial data becomes publicly available at different calendar dates. This leaks future information into the industry mean.

Using `jdate` (the calendar month when the characteristic is available to investors) ensures the cross-section is formed at a common point in time, consistent with how `bm_ia` and `me_ia` were already fixed.

**Note**: Pre-merge `_ia` variables (herf, chempia, chpmia, chatoia at L1043-1104) correctly use `datadate` because at that stage each row is one fiscal period — no monthly expansion yet.

**Reference**: Daniel & Titman (2006), "Market Reactions to Tangible and Intangible Information", JF. GHZ replication convention.

## Fix 6: Quarterly `pm` — DuPont Identity Broken

**Location**: ~Line 1826

**Before**: `ttm4('oiadpq', data_rawq) / pl.col('saleq').replace(0, None)`
**After**: `ttm4('oiadpq', data_rawq) / pl.col('saleq4').replace(0, None)`

The Soliman (2008) DuPont decomposition requires:
```
RNA = PM × ATO
```

With TTM numerators and 4-quarter-lagged NOA:
- `RNA = TTM(oiadpq) / noa_raw_l4`
- `ATO = TTM(saleq) / noa_raw_l4` (= `saleq4 / noa_raw_l4`)
- `PM = TTM(oiadpq) / TTM(saleq)` (= `TTM(oiadpq) / saleq4`)

The previous code used single-quarter `saleq` in PM's denominator:
```
PM_old × ATO = [TTM(oiadpq)/saleq] × [saleq4/noa_raw_l4] ≠ RNA
```

Fix: use `saleq4` (= `ttm4('saleq')`, already computed) as PM's denominator:
```
PM_new × ATO = [TTM(oiadpq)/saleq4] × [saleq4/noa_raw_l4] = TTM(oiadpq)/noa_raw_l4 = RNA ✓
```

**Reference**: Soliman (2008), "The Use of DuPont Analysis by Market Participants", TAR.
